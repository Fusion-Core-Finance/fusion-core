# fusd-spec v1.0.0

> **Implementation-independent formal specification of the fusion-core (FUSD) protocol.**
> This document enumerates every persisted account (a *stage*) and every production
> instruction (an *action*) — with its preconditions, exact arithmetic, guards, and
> post-state — cross-linked to `programs/fusd-core/src/**` (and the `fusd-math` /
> `fusd-oracle` crates) at `file:line`, so a reviewer can grep-compare the spec against
> the implementation line by line. It is the auditor-grade companion to the narrative
> [`docs/fusion-docs.md`](fusion-docs.md): where that reference explains *why*, this one
> pins *what the code does*. Every field, formula, guard, and invariant below was written
> against, and adversarially verified against, the actual source.
>
> **Scope & version.** `v1.0.0` covers the 52 production instructions and 16 persisted
> account types on `master` (the dev-only `dev_set_price` is out of scope — it is gated
> behind the `dev-oracle` feature and absent from the production build). This revision
> reflects the 2026-07-08 redemption change: `redeem` and `urgent_redeem` pay collateral
> at `mid = (spot + debt_spot) / 2`, and the C9 dynamic redemption fee is charged
> *post-bump*. Changes are logged in [`docs/fusd-spec-CHANGES.md`](fusd-spec-CHANGES.md).
> Line numbers are pinned to the commit that introduced this file; re-verify after any
> edit to the cited handler.

## 1. Notation

This section fixes the notation used throughout the spec. fUSD carries no floating point anywhere: every quantity is an unsigned fixed-point integer, and every value scaled by one of three decimal factors (WAD / RAY / RAD). The rounding direction is chosen *against the protocol* at every step so that accumulated dust can never leave the system under-collateralized. Part (a) defines the scales and the arithmetic discipline; part (b) is the authoritative symbol → account-field map.

### (a) Fixed-point scales and rounding discipline

**The three scales** (module doc, `crates/fusd-math/src/lib.rs:3`–`6`):

| Scale | Value | Const | Measures |
|---|---|---|---|
| **WAD** | `1e18` | `crates/fusd-math/src/lib.rs:38` | token quantities (fUSD-native amounts, collateral, stakes) |
| **RAY** | `1e27` | `crates/fusd-math/src/lib.rs:40` | rates, prices, and the interest accumulator (`spot`/`debt_spot` are RAY-scaled fUSD-native per native collateral unit — `programs/fusd-core/src/state/market.rs:34`) |
| **RAD** | `1e45` | *documented only*, `crates/fusd-math/src/lib.rs:6` | conceptual `art[wad]·rate[ray]` internal balance |

RAD is dimensional documentation, not a materialized constant: there is no `pub const RAD` in the codebase. fUSD stores present-value debt directly (`recorded_debt`, `agg_recorded_debt`) rather than a normalized `art × rate` product, so no RAD-scale balance is ever held on-chain — the `present_debt = art·rate/RAY` realization lands back in WAD-scale fUSD-native units (`crates/fusd-math/src/lib.rs:207`–`212`). Auxiliary halves and the bps denominator: `HALF_WAD`/`HALF_RAY` at `crates/fusd-math/src/lib.rs:41`–`42`, `BPS_DENOMINATOR = 10_000` at `crates/fusd-math/src/lib.rs:44`.

**No floats, U256 intermediates.** Every scaled multiply/divide goes through an exact 256-bit intermediate (`bnum::types::U256`, imported `crates/fusd-math/src/lib.rs:22`). The core `wide_mul_div` forms the full `U256` product `a·b` before dividing (`crates/fusd-math/src/lib.rs:68`–`71`), so a `RAY·RAY = 1e54` product — which overflows `u128` — never wraps. **Multiply-before-divide** is structural: the product is taken at full 256-bit width first, and only the final quotient is narrowed back to `u128`.

**Fail-closed, never truncate.** `mul_div_floor` (`crates/fusd-math/src/lib.rs:85`) and `mul_div_ceil` (`crates/fusd-math/src/lib.rs:92`) return `None` — not a wrapped value — when `denom == 0` or the quotient exceeds `u128::MAX`. Callers map `None` to a revert, so overflow halts the instruction rather than corrupting an accumulator.

**Rounding against the protocol** (policy stated at `crates/fusd-math/src/lib.rs:12`–`15`):
- **Debt rounds UP.** Debt-increasing results use the `_ceil` variant: `wad_mul_up` (`crates/fusd-math/src/lib.rs:180`–`183`), `ray_mul_up` (`crates/fusd-math/src/lib.rs:196`–`199`), and `present_debt = ray_mul_up(art, rate)` (`crates/fusd-math/src/lib.rs:210`–`211`), which rounds a borrower's present-value debt up (`ceil(art·rate/RAY)`).
- **Collateral credit rounds DOWN.** Collateral-out / debt-decreasing results floor: `mul_div_floor`, `wad_mul`/`wad_div` (`crates/fusd-math/src/lib.rs:177`,`187`), `ray_mul`/`ray_div` (`crates/fusd-math/src/lib.rs:193`,`203`). Redemption collateral, e.g., is removed at `mul_div_floor(redeem_amt, RAY, mid)` (`programs/fusd-core/src/instructions/redeem.rs:200`).

**Interest accumulator** `ray_pow` (Maker-style `rpow`, binary exponentiation, `crates/fusd-math/src/lib.rs:218`–`230`) floors each squaring/multiply, so the RAY-scaled power is a *lower* bound on the true value — conservative for an accumulator — and fails closed (`None`) on any intermediate `u128` overflow.

## 2. Stages (accounts)

### `ProtocolConfig` — PDA `[b"config"]` (`CONFIG_SEED`, `constants.rs:8`)
- **Kind:** Borsh `#[account]` (`programs/fusd-core/src/state/protocol_config.rs:7`). SPACE pinned = `8 + (32 * 8) + 1 + 32` (`protocol_config.rs:68`) — 8 disc + 8 Pubkeys + 1 bump + 32 reserve.
- **Fields:**
  - `gov_authority: Pubkey` — migratable inbound governance authority (MetaDAO Squads vault PDA); sole writer of bounded params (`protocol_config.rs:13`).
  - `guardian: Pubkey` — de-risk-only, independent of futarchy/Squads (`protocol_config.rs:16`).
  - `deployer: Pubkey` — who ran `init_protocol` (informational) (`protocol_config.rs:18`).
  - `fusd_mint: Pubkey` — the fUSD mint (legacy SPL, freeze=None, mint authority = PDA) (`protocol_config.rs:20`).
  - `bump: u8` — canonical `[b"config"]` bump (`protocol_config.rs:22`).
  - `pending_gov_authority: Pubkey` — two-step handoff successor; `Pubkey::default()` = none in flight. **Carved from the head of `_reserved`** (`protocol_config.rs:29`).
  - `pyth_receiver_program_id: Pubkey` — primary Pyth receiver that must own every `PriceUpdateV2`; NEVER `Pubkey::default()`. **Carved from `_reserved`** (`protocol_config.rs:36`).
  - `pyth_receiver_program_id_alt: Pubkey` — second accepted Pyth receiver (dual-fetch cutover); `Pubkey::default()` = disabled. **Carved from `_reserved`** (`protocol_config.rs:45`).
  - `switchboard_program_id: Pubkey` — Switchboard On-Demand program that must own every `PullFeedAccountData`. **Carved from `_reserved`** (`protocol_config.rs:50`).
  - `_reserved: [u8; 32]` — forward-compat reserve; deliberately NO global emergency flag (`protocol_config.rs:59`).
- **Init:** `init_protocol` — `#[account(init, …, seeds = [CONFIG_SEED])]` at `programs/fusd-core/src/instructions/init_protocol.rs:43`; fields written at `init_protocol.rs:83-98`. Bootstrap gated to the program's upgrade authority (`init_protocol.rs:34-40`).
- **Account-local invariants:** `gov_authority != Pubkey::default()` enforced at init — `require!(args.gov_authority != Pubkey::default(), FusdError::ParamOutOfBounds)` (`init_protocol.rs:81`); a default gov key would brick every gov-gated bootstrap. `pyth_receiver_program_id` is never zero (a zero program ID bricks the crank; `protocol_config.rs:34-35`).

### `fUSD mint` — PDA `[b"fusd_mint"]` (`FUSD_MINT_SEED`, `constants.rs:55`)
- **Kind:** SPL `anchor_spl::token::Mint` (legacy SPL Token, NOT a program `#[account]` struct; NOT Token-2022) — `init_market.rs`/`init_protocol.rs` bind `Program<Token>` at `init_protocol.rs:70`. Space is the SPL mint layout (Anchor `init` sizes it).
- **Fields:** standard SPL mint — `mint_authority = mint_authority` PDA (`mint::authority = mint_authority`, `init_protocol.rs:66`), `decimals = FUSD_DECIMALS = 6` (`mint::decimals = FUSD_DECIMALS`, `init_protocol.rs:65`; `FUSD_DECIMALS` = 6 at `init_protocol.rs:10`), `supply` (grows/shrinks only via PDA-signed mint/burn), `freeze_authority = None` (freeze arg omitted ⇒ `None`, irreversible — the censorship-resistance guarantee, `init_protocol.rs:57-67`).
- **Init:** `init_protocol` — `#[account(init, payer, seeds = [FUSD_MINT_SEED], …)]` at `programs/fusd-core/src/instructions/init_protocol.rs:60-68`.
- **Account-local invariants:** `freeze_authority == None` (permanent — no field can re-enable it) and `mint_authority == PDA[b"mint_authority"]` by construction (`init_protocol.rs:60-67`); `decimals == 6` fixed at init.

### `mint_authority` PDA — PDA `[b"mint_authority"]` (`MINT_AUTHORITY_SEED`, `constants.rs:53`)
- **Kind:** none — a pure PDA signer, declared as `UncheckedAccount` with `seeds`-only (`#[account(seeds = [MINT_AUTHORITY_SEED], bump)]`, `programs/fusd-core/src/instructions/init_protocol.rs:54-55`). It is NEVER a keypair and holds NO account data; minting/burning is possible only via `invoke_signed` under the rules (`init_protocol.rs:52-53`).
- **Fields:** none (no backing account is allocated — no discriminator, no rent, no state).
- **Init:** NOT initialized as an account — it is derived (never created) wherever it signs; first referenced at `programs/fusd-core/src/instructions/init_protocol.rs:54` as the fUSD mint's authority.
- **Account-local invariants:** none of its own (no fields). Cross-account: it is the sole minting authority, so `fusd_mint.mint_authority == PDA[b"mint_authority"]` (`init_protocol.rs:66`).

### `provide_to_reactor`
- **Caller:** Signer `owner`, permissionless. Must match the deposit's stored owner via `has_one = owner` and be the fUSD ATA authority (`programs/fusd-core/src/instructions/provide_to_reactor.rs:13`, `programs/fusd-core/src/instructions/provide_to_reactor.rs:32`, `programs/fusd-core/src/instructions/provide_to_reactor.rs:36`).
- **Preconditions:** `ReactorPool` (mut), `ReactorDeposit` (mut, owner-bound), and the zero-copy `EpochToScaleToSum` grid must exist; the grid account must be exactly `reactor_pool.epoch_to_scale_to_sum` and the vault exactly `reactor_pool.fusd_vault` (`programs/fusd-core/src/instructions/provide_to_reactor.rs:24-26`, `programs/fusd-core/src/instructions/provide_to_reactor.rs:39`). `owner_fusd_ata` must hold ≥ `amount` fUSD (enforced by the SPL transfer CPI).
- **Formula:** Notation — recorded `d₀ = deposited_fusd`; snapshot `{P₀,S₀,scale₀,epoch₀}`; pool `{P, S-grid, scale, epoch, total_deposits}`; `DECIMAL_PRECISION = 1e18`, `SCALE_FACTOR = 1e9`.
  1. **Realize** (`reactor::realize`, `programs/fusd-core/src/reactor.rs:40`): compute the compounded deposit `c` = `compounded_deposit` (`crates/fusd-math/src/reactor_pool.rs:289`): `c = 0` if `d₀ = 0` or `epoch₀ < epoch` (a full-pool drain since the snapshot wiped it); else with `Δ = scale − scale₀`: `Δ=0 → c = floor(d₀·P / P₀)`; `Δ=1 → c = floor(d₀·P / P₀) / SCALE_FACTOR`; `Δ≥2 → c = 0`; then the dust floor `c < d₀ / 1e9 → c = 0` (`crates/fusd-math/src/reactor_pool.rs:294-306`). Accrued gain `g` = `collateral_gain` (`crates/fusd-math/src/reactor_pool.rs:311`): `first = S[epoch₀,scale₀] − S₀`, `second = S[epoch₀,scale₀+1] / SCALE_FACTOR`, `g = floor(d₀·(first+second) / P₀) / DECIMAL_PRECISION` (floored — collateral credit DOWN). `pending_collateral_gain += g` (`programs/fusd-core/src/reactor.rs:46`).
  2. **Re-snapshot** to the current pool point (provide leaves `P/scale/epoch` unchanged): `{P, S[epoch,scale], scale, epoch}` (`programs/fusd-core/src/reactor.rs:54`, `crates/fusd-math/src/reactor_pool.rs:125`).
  3. `new_deposited = c + amount` (checked_add, then `u64::try_from`) (`programs/fusd-core/src/instructions/provide_to_reactor.rs:57-60`).
  4. Transfer `amount` fUSD `owner_fusd_ata → reactor_fusd_vault` (`programs/fusd-core/src/instructions/provide_to_reactor.rs:62`).
  5. `total_deposits += amount` (checked_add); `deposited_fusd = new_deposited` (`programs/fusd-core/src/instructions/provide_to_reactor.rs:74-80`).
- **Guards:** `require!(amount > 0, FusdError::ZeroAmount);` (`programs/fusd-core/src/instructions/provide_to_reactor.rs:46`). `has_one = owner` on the deposit (`programs/fusd-core/src/instructions/provide_to_reactor.rs:32`); `address = reactor_pool.epoch_to_scale_to_sum` on the grid (`programs/fusd-core/src/instructions/provide_to_reactor.rs:25`); `address = reactor_pool.fusd_vault` on the vault (`programs/fusd-core/src/instructions/provide_to_reactor.rs:39`); `token::mint = fusd_mint, token::authority = owner` on the source ATA (`programs/fusd-core/src/instructions/provide_to_reactor.rs:36`). Overflow of `c + amount` or `total_deposits + amount` returns `FusdError::MathOverflow` (`programs/fusd-core/src/instructions/provide_to_reactor.rs:58`, `programs/fusd-core/src/instructions/provide_to_reactor.rs:79`); a grid-index/scale/epoch overflow inside `collateral_gain` maps to `FusdError::ReactorGridExhausted`/`MathOverflow` (`programs/fusd-core/src/reactor.rs:62-67`).
- **Post-state:** `ReactorDeposit`: `pending_collateral_gain` increased by `g`; snapshot rolled forward; `deposited_fusd = c + amount`. `ReactorPool`: `total_deposits += amount`. `reactor_fusd_vault` balance `+= amount`. Emits `ReactorDepositUpdated{op=PROVIDE, fusd_amount=amount, collateral_paid=0, deposited_fusd=new_deposited}` (`programs/fusd-core/src/instructions/provide_to_reactor.rs:82-89`).
- **Code:** `programs/fusd-core/src/instructions/provide_to_reactor.rs:45` (handler entry); `programs/fusd-core/src/reactor.rs:40` (realize), `programs/fusd-core/src/reactor.rs:54` (set_snapshot); `crates/fusd-math/src/reactor_pool.rs:289` (compounded_deposit), `crates/fusd-math/src/reactor_pool.rs:311` (collateral_gain), `crates/fusd-math/src/reactor_pool.rs:125` (snapshot).

### `liquidate`

- **Caller:** Permissionless. `liquidator: Signer` with no authority check (`programs/fusd-core/src/instructions/liquidate.rs:31`); it receives the SOL reserve bond plus the collateral gas-comp. There is deliberately **no** `shutdown` guard — a tier-4 liquidation in an already-shut market re-books bad debt without re-announcing shutdown (`programs/fusd-core/src/instructions/liquidate.rs:456`).
- **Preconditions:** `market`, `position` (`has_one = collateral_mint`), `reactor_pool`, its `epoch_to_scale_to_sum` grid, the market/RP/buffer vaults, `fusd_mint`, `redemption_bitmap`, and the per-market `insurance_buffer` (+ its fUSD vault) are all required every call (`programs/fusd-core/src/instructions/liquidate.rs:34`-`77`). `liquidator_collateral_ata` is pinned `token::authority = liquidator` so the gas-comp skim can't alias a program vault (`programs/fusd-core/src/instructions/liquidate.rs:64`). The Global Backstop reserve + its vault are **optional** (`Option<Box<…>>`); omitted ⇒ `global_available == 0`, a pre-backstop 4-tier waterfall (`programs/fusd-core/src/instructions/liquidate.rs:84`-`88`). Oracle must be fresh; the position must be strictly below MCR at the HIGH price.
- **Formula:** Prices off the **HIGH** price: `liq_spot = debt_spot = mid + k·σ`, never the LOW `spot` — optimism protects against destroying a position on noise (`programs/fusd-core/src/instructions/liquidate.rs:104`). `accrue` then `realize` the victim → `debt = recorded_debt`, `ink` (`programs/fusd-core/src/instructions/liquidate.rs:95`,`135`-`138`).
  - Eligibility (inverse of health): unhealthy ⇔ `debt > max_debt(collateral_value(ink, liq_spot), mcr_bps)` where `collateral_value = ink·liq_spot/RAY` (floor) and `max_debt = value·10_000/mcr_bps` (floor) (`programs/fusd-core/src/cdp.rs:48`,`11`,`16`).
  - **Collar** `seize_collateral(ink, debt, liq_spot, liq_bonus_bps)`: `seize_value = ceil(debt·(10_000+liq_bonus_bps)/10_000)`; `seize_coll = min(ceil(seize_value·RAY/liq_spot), ink)`; `surplus_coll = ink − seize_coll`. Both conversions round **UP** against the borrower; `liq_bonus_bps == 0` ⇒ seize all (`programs/fusd-core/src/cdp.rs:34`-`43`).

### `redeem`

- **Caller:** Permissionless. `redeemer: Signer` (`programs/fusd-core/src/instructions/redeem.rs:26`). Passes candidate `Position` accounts as `remaining_accounts` (writable, in the lowest non-empty bucket).
- **Preconditions:** market **not** shut down (a shut market winds down via `urgent_redeem`); `amount > 0`; candidate count within cap; fresh oracle with both price legs present; something to redeem in the lowest normal bucket or the zombie pen. `redeemer_fusd_ata`/`redeemer_collateral_ata` pinned to the signer (`programs/fusd-core/src/instructions/redeem.rs:43`-`47`).
- **Formula:** `accrue` the market, then pay at the **MID** price `mid = (spot + debt_spot)/2` — NOT the LOW `spot` (dividing an outflow by the LOW price over-pays collateral at peg) (`programs/fusd-core/src/instructions/redeem.rs:75`). `lowest_normal = first_set(words)` (find-first-set — can't skip a lower bucket) (`programs/fusd-core/src/instructions/redeem.rs:109`; `crates/fusd-math/src/rate_bucket.rs:55`). Candidates are validated (each `realize`d + `reweight`ed + `set_stake`d), deduped, and sorted **lowest-CR-first** among the submitted set via `cmp_collateral_ratio` (`ink_a·art_b` vs `ink_b·art_a` in 256-bit) (`programs/fusd-core/src/instructions/redeem.rs:176`; `crates/fusd-math/src/rate_bucket.rs:96`). Per candidate:
  - `coll_value = ray_mul(ink, mid) = ink·mid/RAY`; `redeem_amt = min(remaining, recorded_debt, coll_value)` (never creates bad debt); `coll_total = min(floor(redeem_amt·RAY/mid), ink)` (`programs/fusd-core/src/instructions/redeem.rs:191`-`202`).
  - `recorded_debt −= redeem_amt`; `ink −= coll_total`; `agg_recorded_debt −= redeem_amt`; `total_collateral −= coll_total`; `coll_drawn_total += coll_total` (`:208`-`241`).
  - `redeemed_fusd = amount − remaining` (`:244`).
  - **C9 fee is charged POST-bump:** `decayed_base_rate = decay_base_rate(redemption_base_rate, now − redemption_base_rate_ts)` decayed up front (`:85`); after the loop `new_base_rate = bump_base_rate(decayed_base_rate, redeemed_fusd, total_debt_before) = decayed + floor((redeemed_fusd·RAY/agg_recorded_debt_before)/BETA)`, saturating at `RAY` (`:251`; `crates/fusd-math/src/redemption.rs:47`). Only THEN `fee_bps = effective_fee_bps(new_base_rate, redemption_fee_bps, redemption_base_rate_max_bps, MAX_REDEMPTION_FEE_BPS) = min(redemption_fee_bps + min(base_rate→bps, redemption_base_rate_max_bps), 500)` (`:260`; `crates/fusd-math/src/redemption.rs:65`; `programs/fusd-core/src/constants.rs:252`). `total_debt_before` is the pre-redeem post-accrue `agg_recorded_debt` (`:84`).
  - `fee_coll_total = floor(coll_drawn_total·fee_bps/10_000)` (retained as surplus, rounds UP vs the old per-position floor); `redeemer_coll_total = coll_drawn_total − fee_coll_total` (`:269`-`270`).
- **Targeting is health-INDEPENDENT (by design):** redemption selects by rate bucket (`Position.user_rate_bps`), never by collateral ratio — a well-collateralized position at the lowest rate is redeemed before an underwater one at a higher rate, and a high CR does **not** protect a borrower. This is correct: the peg floor must always reach *someone*, and rate-keying keeps it oracle-independent. Being redeemed is value-neutral (debt cancelled at face, collateral paid at `mid`), not a loss. A borrower's only exposure lever is raising `user_rate_bps` via `adjust_rate` to sit behind lower-rate troves (deterred from gaming by the premature-adjust fee). There is deliberately no redemption opt-out.
- **Guards:** verbatim:

  - `require!(!ctx.accounts.market.shutdown, FusdError::MarketShutdown)` (`:62`)

### `urgent_redeem`

- **Caller:** Permissionless. `redeemer: Signer` (`programs/fusd-core/src/instructions/urgent_redeem.rs:28`). Candidates passed as `remaining_accounts` from **any** bucket (unordered wind-down).
- **Preconditions:** market **must** be `shutdown`; `amount > 0`; candidate count within cap; a price must exist (both legs `> 0`) but there is **no staleness gate** (a market shut on oracle failure still redeems at its last good price) (`programs/fusd-core/src/instructions/urgent_redeem.rs:60`,`72`-`76`).
- **Formula:** `accrue` is a no-op while shut (interest frozen at the wind-down moment) (`programs/fusd-core/src/instructions/urgent_redeem.rs:69`; `programs/fusd-core/src/accrual.rs:31`,`59`). Pays at the **MID** `mid = (spot + debt_spot)/2` — same as ordered `redeem`, never the conservative LOW `spot` (`programs/fusd-core/src/instructions/urgent_redeem.rs:77`). Per candidate (any position, unordered, **0-fee**): `realize`; `coll_value = ray_mul(ink, mid) = ink·mid/RAY`; `redeem_amt = min(remaining, recorded_debt, coll_value)`; `coll_total = min(floor(redeem_amt·RAY/mid), ink)` (`:115`-`133`). Then `recorded_debt −= redeem_amt`, `ink −= coll_total`, `agg_recorded_debt −= redeem_amt`, `total_collateral −= coll_total`, `redeemer_coll_total += coll_total` (full amount — no skim) (`:137`-`168`). `redeemed_fusd = amount − remaining` (`:171`).
- **Guards:** verbatim:
  - `require!(amount > 0, FusdError::ZeroAmount)` (`:59`)
  - `require!(ctx.accounts.market.shutdown, FusdError::MarketNotShutdown)` (`:60`)
  - `require!(ctx.remaining_accounts.len() <= MAX_REDEMPTION_CANDIDATES, FusdError::TooManyRedemptionCandidates)` (`:62`)
  - `require!(spot > 0 && debt_spot > 0, FusdError::OracleUnavailable)` (`:76`)
  - Per candidate: skip closed `if info.owner != &crate::ID || info.data_is_empty() { continue; }` (`:92`); `require!(!seen.contains(&key), FusdError::DuplicateRedemptionTarget)` (`:96`); `require_keys_eq!(pos.collateral_mint, coll_mint, FusdError::Unauthorized)` (`:100`); `if redeem_amt == 0 { … continue; }` (fully reconciles then skips) (`:118`).
  - `require!(redeemed_fusd > 0, FusdError::NothingToRedeem)` (`:172`). No staleness / CR-sort / bucket-order / fee (the deliberate differences from `redeem`). Caps `min(remaining, debt, coll_value)` and `min(…, ink)` prevent bad debt / over-draw (`:117`,`131`).

Tokens: burn `redeemed_fusd`; transfer `redeemer_coll_total` from vault → redeemer; vault-sufficiency reconcile (`:174`-`212`).

### `shutdown`

- **Caller:** Permissionless. `cranker: Signer` signs only to carry the tx — **no authority check**, no guardian/gov key (the one lever that permanently closes a market is condition-gated only, so it can't become a coercible kill switch) (`programs/fusd-core/src/instructions/shutdown.rs:28`-`29`).
- **Preconditions:** market not already `shutdown`; `accrue` folds aggregate interest so the TCR check sees current present debt (`programs/fusd-core/src/instructions/shutdown.rs:38`-`42`).
- **Formula:** `staleness = slot − spot_updated_slot` (`:46`). Two condition-gated triggers:
  - **Oracle failure:** `oracle_failed = spot > 0 && staleness > SHUTDOWN_ORACLE_STALENESS_SLOTS` (`= 9_000`) — a never-priced market (`spot == 0`) is pre-launch, not failed (`:51`; `programs/fusd-core/src/constants.rs:163`).
  - **SCR breach** (assessed only on a fresh price): `tcr_breach = spot > 0 && staleness <= MAX_PRICE_STALENESS_SLOTS && tcr_below(total_collateral, agg_recorded_debt, spot, scr_bps)`, where `tcr_below` ⇔ `agg_recorded_debt > max_debt(collateral_value(total_collateral, spot), scr_bps) = (total_collateral·spot/RAY)·10_000/scr_bps`, and returns `false` on zero debt or any arithmetic edge (a math overflow can never spuriously trigger shutdown) (`:55`-`57`; `programs/fusd-core/src/cdp.rs:66`). Note this uses the conservative **LOW** `spot` (SCR/CCR keep the pessimistic price).
  - `reason = if oracle_failed { SHUTDOWN_REASON_ORACLE_FAILURE (2) } else { SHUTDOWN_REASON_SCR (1) }` — oracle failure takes precedence when both hold (`:62`; `programs/fusd-core/src/constants.rs:168`-`169`).
- **Guards:** verbatim:
  - `require!(!ctx.accounts.market.shutdown, FusdError::MarketShutdown)` (`:38`)
  - `require!(oracle_failed || tcr_breach, FusdError::ShutdownConditionNotMet)` (`:59`)
- **Post-state:** `market.shutdown = true` (irreversible; only remedy is migration) and `market.shutdown_reason = reason` (`:63`-`64`); emits `ShutdownEvent`. This closes `borrow` and ordered `redeem` and opens the 0-fee unordered `urgent_redeem`.
- **Code:** `programs/fusd-core/src/instructions/shutdown.rs:37` (handler); `programs/fusd-core/src/cdp.rs:66` (`tcr_below`); `programs/fusd-core/src/accrual.rs:29` (`accrue`).

### `execute_param_change`
- **Caller:** Permissionless — any signer `executor`, who also receives the closed op's rent (`programs/fusd-core/src/instructions/governance.rs:510`). The gate is not read here; the timelock is baked into `op.eta` at queue time (no `gov_gate` in the context).

Monotone `max` never shortens an active grace; only raises arm it (a lowering is pure de-risk). Compared BEFORE `apply_param` so it reads the pre-change `mcr_bps`. `prev_value` is captured via `current_param` immediately before apply for the forensic trail (`programs/fusd-core/src/instructions/governance.rs:578`, reader `:187`).

### `bump_debt_ceiling`
- **Caller:** **Permissionless.** `cranker: Signer` signs only to carry the tx; there is no authority check (`programs/fusd-core/src/instructions/debt_ceiling_line.rs:148-149`). Safe to open because the bump can never push `market.debt_ceiling` past the gov-set `line` — it only performs throttled, bounded follow-the-utilization motion within `[agg_recorded_debt, line]` (`state/debt_ceiling_line.rs:9-13`).
- **Preconditions:** The `DebtCeilingLine` PDA must exist (`#[account(mut, seeds=[RATE_LIMITER_SEED, collateral_mint], bump=debt_ceiling_line.bump)]`, `programs/fusd-core/src/instructions/debt_ceiling_line.rs:156-161`) and the `market` PDA must exist (`:153`). - **Guards:**
  - `require!(now >= last.saturating_add(dcl_ttl), FusdError::RateLimitExceeded)` — the DC-IAM once-per-`ttl` throttle, where `last = debt_ceiling_line.last_bump_ts`, `dcl_ttl = debt_ceiling_line.ttl` (`programs/fusd-core/src/instructions/debt_ceiling_line.rs:169-171`).
  - No authority guard (intentional, `:148-149`). Accrue may abort with `FusdError::MathOverflow` (`programs/fusd-core/src/accrual.rs:36,38,40`).

  - `backing = sum_agg_plus_bad − sum_unminted` (saturating, non-negative in aggregate) (`supply_reconciliation.rs:109`)
  - `mint_supply = fusd_mint.supply as u128` (`supply_reconciliation.rs:110`)
  - `residual = (backing as i128) − (mint_supply as i128)` (signed; `0` = reconciled, `+` ⇒ markets over-account / double-count, `−` ⇒ mint over-shows / market omitted) (`supply_reconciliation.rs:112`, semantics `state/supply_reconciliation.rs:26-29`)
- **Guards:**
  - `require!(ctx.remaining_accounts.len() <= MAX_RECONCILE_MARKETS, FusdError::TooManyRedemptionCandidates)` — `MAX_RECONCILE_MARKETS = 24` (`supply_reconciliation.rs:80-83`, `constants.rs:62`).
  - Per market (program-owned): `require_keys_eq!(*info.owner, crate::ID, FusdError::InvalidRecipient)` (`supply_reconciliation.rs:92`).
  - Per market (dedupe, so the sum can't be inflated): `require!(!seen.contains(&key), FusdError::DuplicateRedemptionTarget)` (`supply_reconciliation.rs:94`).
  - Per market (discriminator): `Account::<Market>::try_from(info)?` verifies the `Market` discriminator; a non-`Market` account reverts (`supply_reconciliation.rs:96`).
  - Both sums fail closed on overflow: `.checked_add(...).ok_or(FusdError::MathOverflow)?` (`supply_reconciliation.rs:98-104`).
  - `backing` uses `saturating_sub`; `residual` uses `(backing as i128).saturating_sub(mint_supply as i128)` — no revert path here (`supply_reconciliation.rs:109,112`).
  - NO guard on the result: the handler NEVER reverts on a non-zero `residual` — the residual is the signal, and reverting would blind the monitor when drift exists (`supply_reconciliation.rs:76-78`).
- **Post-state:** Writes only the `SupplyReconciliation` singleton (moves no funds, gates nothing) (`supply_reconciliation.rs:115-120`):
  - `last_ts = Clock::get()?.unix_timestamp`
  - `last_market_count = remaining_accounts.len() as u32`
  - `last_mint_supply = mint_supply`
  - `last_backing = backing`
  - `last_residual = residual`
  - Emits `SupplyReconciled { market_count, mint_supply, backing, residual }` (`supply_reconciliation.rs:122-127`, event def `events.rs:425-430`).
- **Code:** `programs/fusd-core/src/instructions/supply_reconciliation.rs:79` (handler `reconcile`, registered as `reconcile_supply` at `programs/fusd-core/src/lib.rs:205-208`). The reconciliation arithmetic is entirely inline — sums at `supply_reconciliation.rs:98-104`, `backing`/`residual` at `supply_reconciliation.rs:109-112`. This crank does NOT call `reconcile.rs`: that module (`programs/fusd-core/src/reconcile.rs:37-66`, `assert_collateral_vault_sufficiency` / `check_sufficiency`) is the per-handler `vault >= Σ tracked` collateral-vault sufficiency check invoked by fund-moving handlers, a distinct proof-of-reserves layer with no call edge to `reconcile_supply`.

## 4. Invariants

### 1. fUSD mint carries no freeze authority (`freeze_authority == None`)

**Statement.** The fUSD mint is created with `freeze_authority == None`, irreversibly — no authority (governance, guardian, or otherwise) can ever freeze a holder's fUSD. The core censorship-resistance guarantee.

**Enforcement.** Structural, at the mint's `init` in `programs/fusd-core/src/instructions/init_protocol.rs:60-68`: the `#[account(init, …)]` block sets `mint::authority = mint_authority` and **omits** `mint::freeze_authority`, so Anchor/SPL creates the mint with `freeze_authority = None` (documented at `programs/fusd-core/src/instructions/init_protocol.rs:57-59`). There is no `require!` — the guarantee is that the field is never populated, and SPL makes `None` permanent. (Separately, the analogous check on *collateral* mints is a runtime guard, `require!(ctx.accounts.collateral_mint.freeze_authority.is_none(), FusdError::CollateralHasFreezeAuthority);` at `programs/fusd-core/src/instructions/init_market.rs:89-92` — a different invariant.)

**Formal verification.** Enforced only (no FV artifact). It is a one-time structural Anchor constraint at genesis, not a cross-instruction state-transition property, so there is no Kani harness or Certora rule.

### 2. fUSD mint authority is the `[b"mint_authority"]` PDA (no off-chain key)

**Statement.** The only signer that may `mint_to` fUSD is the program-derived `[b"mint_authority"]` PDA; there is no keypair mint authority. Every mint therefore happens only via `invoke_signed` from inside a handler executing under the rules. (Burns are SPL-authorized by the fUSD holder — the token-account owner: the borrower on `repay`, or a program PDA such as `market`/`reactor_pool`/`insurance_buffer`/backstop on `redeem`/`liquidate`/`urgent_redeem`/`settle_bad_debt` — never by this PDA; the mint authority governs minting only.)

**Enforcement.** Structural + runtime signer:
- Mint authority is bound to the PDA at genesis — `programs/fusd-core/src/instructions/init_protocol.rs:66` (`mint::authority = mint_authority`) where `mint_authority` is the PDA `#[account(seeds = [MINT_AUTHORITY_SEED], bump)]` at `programs/fusd-core/src/instructions/init_protocol.rs:52-55`. The seed constant is `MINT_AUTHORITY_SEED = b"mint_authority"` at `programs/fusd-core/src/constants.rs:52-53`.
- Every mint signs only under the PDA seeds: `programs/fusd-core/src/instructions/borrow.rs:170-183` builds `signer = &[&[MINT_AUTHORITY_SEED, &[bump]]]` and passes it to `token::mint_to` (mirror `mint_to` sites in `programs/fusd-core/src/instructions/refresh_market.rs:129-171`; the same PDA also signs the metadata `invoke_signed` at `create_fusd_metadata.rs:197`).

**Formal verification.** Enforced only (no FV artifact). Authority binding and PDA signing are Anchor/SVM constraints, not a symbolic-state property; no Kani/Certora artifact.

### 3. 4-term collateral-vault sufficiency

**Statement.** `vault >= total_collateral + surplus_collateral + total_coll_surplus + protocol_collateral` — the collateral vault's token balance is at least the sum of the four ledger counters (proof-of-reserves). Direction is load-bearing: the on-chain check is **sufficiency (`>=`)**, never exact `==`; the dangerous direction (vault under-funds the ledger) hard-reverts in the same tx, while a permissionless over-donation passes (an absolute `==` would let a 1-lamport donation brick liquidation/redemption with no admin recourse).

**Enforcement.** `programs/fusd-core/src/reconcile.rs:64`: `require!((vault_amount as u128) >= tracked, FusdError::VaultReconciliationFailed);` where `tracked` is the checked 4-term sum (`programs/fusd-core/src/reconcile.rs:59-63`). Called as the last statement of every collateral-moving handler: `withdraw.rs:155`, `deposit.rs:132`, `redeem.rs:319`, `liquidate.rs:540`, `urgent_redeem.rs:209`, `withdraw_surplus.rs:88`, `sweep_protocol_collateral.rs:92`, `claim_coll_surplus.rs:81` (all under `programs/fusd-core/src/instructions/`).

**Formal verification.** Enforced only (no Kani/Certora FV artifact). The pure comparison `check_sufficiency` is unit-tested (`programs/fusd-core/src/reconcile.rs:72-88`, incl. the checked-sum fail-closed case) and the stricter **exact `==`** form is asserted across the litesvm invariant suite (the runnable oracle), but there is no Kani harness and no rule in `programs/fusd-core/src/certora.rs` / row in `certora/mutations.md`.

### 4. Global supply identity (per market)

**Statement.** `circulating == agg_recorded_debt − unminted_interest + bad_debt` — for each market, the circulating fUSD (SPL mint supply) equals recorded aggregate debt minus not-yet-minted interest plus booked bad debt. Preserved by every supply-touching instruction. (Field-documented at `programs/fusd-core/src/state/market.rs:30`.)

**Enforcement.** Maintained by construction — each supply-touching handler executes its shared supply-transition function (`programs/fusd-core/src/supply_transition.rs`, audit M-01) and commits the returned post-state. **Formal verification.**
- **Certora — VERIFIED (cloud, all eight rules, `certora/supply.conf`; S1 PROD-FN mutation flip cloud-confirmed):** each rule EXECUTES the shared transition fn its handler runs, monomorphized to `NativeInt` — `supply_preserved_by_borrow_ghost` (`programs/fusd-core/src/certora.rs:156`), `_repay_ghost` (`:211`), `_refresh_market_ghost` (`:241`), `_liquidate_ghost` (`:284`), `_redeem_ghost` (`:317`), `_urgent_redeem_ghost` (`:338`), `_settle_bad_debt_ghost` (`:359`), `_book_interest_ghost` (`:382`). ### 5. Aggregate-ceil interest ≥ Σ per-position-floor interest (solvency-by-rounding)

**Statement.** The market's aggregate accrued interest is rounded **UP** while each position's realized interest is rounded **DOWN**, so the minted aggregate is never short of the sum of per-position debts. Both are the ceil/floor of the *same* quantity `debt · rate_bps · dt / INTEREST_DENOM`, so they differ by at most 1 per position (protocol-favoring).

**Enforcement.** Rounding directions in `crates/fusd-math/src/interest.rs`: `pending_aggregate_interest` rounds up via `mul_div_ceil(weighted_sum, dt, INTEREST_DENOM)` at `crates/fusd-math/src/interest.rs:56`; `accrued_interest` rounds down via `mul_div_floor(weighted, period, INTEREST_DENOM)` at `crates/fusd-math/src/interest.rs:69`. Consumed in production at `programs/fusd-core/src/accrual.rs:35` (aggregate mint delta) and `programs/fusd-core/src/accrual.rs:61` (per-position realize).

**Formal verification.**
- **Kani — STRONG** (`crates/fusd-math/kani_audit.tsv`): `aggregate_never_short_of_position` asserts `agg >= pos` and `agg − pos <= 1` (`crates/fusd-math/src/kani_proofs.rs:508-522`); companion `pending_aggregate_one_year_rounds_up` proves the exact `ceil(weighted/10_000)` round-up (`:493-505`), and `accrued_one_year_is_debt_times_rate` the per-position floor (`:479-490`).
- **proptest — B8** `interest` (6 properties, round-up + aggregate-never-short over wide `(debt, rate, period)` vs a U256 reference). No Certora rule (pure-function property; below the cross-instruction tier).

### 6. Per-vault hard solvency (`recorded_debt ≤ ink·spot`)

**Statement.** No live position may carry more `recorded_debt` (realized present-value debt) than the LOW-price value of its collateral: `recorded_debt <= max_debt(collateral_value(ink, spot), mcr_bps)`, with `collateral_value = floor(ink·spot / RAY)` and `max_debt = floor(value·10_000 / mcr_bps)`. Since `mcr_bps >= 10_000`, this implies the hard floor `recorded_debt <= ink·spot` (RAY-scaled). Evaluated at `spot` (the LOW price `mid − k·σ`) and rounded against the borrower (collateral value floored, debt up) — conservative.

**Enforcement.** The predicate `cdp::is_healthy` at `programs/fusd-core/src/cdp.rs:48-57` (built from `collateral_value` at `cdp.rs:11-13` and `max_debt` at `cdp.rs:16-21`), enforced as a hard guard:
- `programs/fusd-core/src/instructions/borrow.rs:107`: `require!(cdp::is_healthy(ink, new_debt, spot, mcr), FusdError::BelowMinCollateralRatio);`
- `programs/fusd-core/src/instructions/withdraw.rs:73` (same predicate) and `adjust_rate.rs:113`;
- liquidation eligibility is its negation — `programs/fusd-core/src/instructions/liquidate.rs:142` requires `!cdp::is_healthy(ink, debt, liq_spot, mcr)`.

**Formal verification.** Enforced only (no dedicated Kani/Certora artifact for the composite predicate). `is_healthy` / `max_debt` / `collateral_value` are covered by unit tests (`programs/fusd-core/src/cdp.rs:142-159`); their arithmetic primitives are Kani-proven independently (`mul_div_floor_matches_reference`, `apply_bps_never_exceeds_notional` in `crates/fusd-math/kani_audit.tsv`), but no harness or Certora rule proves the composite solvency predicate itself.

### 7. Bitmap ↔ counts coupling

**Statement.** In each market's `RedemptionBitmap`, bit `k` is set **iff** `counts[k] > 0` — the words bitfield and the per-bucket member counts never disagree. This is what makes redemption's find-first-set both correct and non-wedgeable.

**Enforcement.** `programs/fusd-core/src/bucket.rs`: `add_member` sets the bit exactly on the empty→non-empty transition (`rb::set` at `bucket.rs:60`, gated by `if bm.counts[bucket] == 0`), and `remove_member` clears it exactly on the non-empty→empty transition (`rb::clear` at `bucket.rs:77`, gated by `if bm.counts[bucket] == 0` post-decrement). Both are driven from the single membership entry point `reconcile` (`bucket.rs:89-120`).

**Formal verification.**
- **Certora — VERIFIED (cloud):** `bitmap_coupling_preserved_by_add_member` (`programs/fusd-core/src/certora.rs:428`) and `bitmap_coupling_preserved_by_remove_member` (`certora.rs:463`), each driving the real `bucket::add_member`/`remove_member`. - **proptest — B8** `rate_bucket` (14 properties, incl. the stateful concurrent-flip `BTreeSet` model). The earlier `bitmap_coherence_preserved_by_reconcile` (`certora.rs:57`) is the retained, *superseded* characterized-frontier attempt (symbolic-index store — spurious CE), not a passing conf.

### 8. Liquidation full-debt conservation + strict tier ordering

**Statement.** A single liquidation routes the entire present `debt` across five loss-absorption tiers with nothing stranded: `reactor + redist + buffer + global + unhomed == debt` (`reactor` = the Reactor-Pool/stability offset, `sp`). Strictly ordered RP → redistribution → local buffer → global backstop → un-homed: a tier contributes only after every higher-priority tier is exhausted and no redistribution recipient exists. A non-zero `unhomed` is the terminal shutdown signal (never lost debt), so the waterfall can never stall.

**Enforcement.** `crates/fusd-math/src/recovery.rs:64-80` — `absorb` is total `min`/`sub`/`add`: `reactor = debt.min(reactor_capacity)` (`:71`) → `redist` all-or-nothing (`:73`) → `buffer = rem.min(buffer_balance)` (`:75`) → `global = rem.min(global_available)` (`:77`) → `unhomed = rem − global` (`:78`), returning `Absorption { reactor, redist, buffer, global, unhomed }` (`:79`). Consumed at `programs/fusd-core/src/instructions/liquidate.rs:223`; on `split.unhomed > 0` the handler books it to `bad_debt` and trips `shutdown` at `liquidate.rs:438-459`.

**Formal verification.**
- **Certora — VERIFIED (cloud):** `absorb_conserves_debt` (`programs/fusd-core/src/certora.rs:502`) asserts `a.reactor + a.redist + a.buffer + a.global + a.unhomed == debt`; mutation **L1** = `let unhomed = 0;` in `recovery.rs` → VIOLATED. `absorb_unhomed_iff_no_tier_covers` (`certora.rs:529`) + non-vacuity witness `absorb_unhomed_reachable` (`certora.rs:555`) prove the strict ordering; mutation **L3** (reorder the global tier ahead of the local buffer) flips `global>0 ⟹ buffer==buffer_balance` to VIOLATED (conf `certora/absorb.conf`).
- **Kani — STRONG** (`crates/fusd-math/kani_audit.tsv`): `absorb_conserves_debt_exactly` (conservation over fully-symbolic u128, `crates/fusd-math/src/kani_proofs.rs:415`) and `absorb_is_fail_closed_and_ordered` (strict order + fail-closed haircuts + `unhomed>0` iff no tier covers, `:436`). **proptest — B8** `recovery` (3).
- The coupled on-chain consequence `unhomed > 0 ⟹ market.shutdown` (`liquidate.rs:438-459`) is **enforced only** — mutations.md row **L2** (`unhomed_debt_always_trips_shutdown`) is authored/not-yet-run; no Certora/Kani artifact yet.

### 9. Redemption starts at the lowest non-empty bucket (cannot skip)

**Statement.** A redemption targets the lowest non-empty NORMAL rate bucket (find-first-set) and can never skip a lower one — the "debt-in-front" ordering. (The zombie pen sits outside this ordering and may be drained out-of-band.)

**Enforcement.** `programs/fusd-core/src/instructions/redeem.rs`: the target is `lowest_normal = rb::first_set(&bm.words)` at `redeem.rs:107-109`, gated by `require!(lowest_normal.is_some() || has_zombies, FusdError::NothingToRedeem);` (`redeem.rs:111`). Each candidate is accepted only if it is in that bucket (or a zombie), else skipped — `redeem.rs:144-147`: `let b = pos.bucket as usize; if !(pos.recorded_debt > 0 && (b == ZOMBIE_BUCKET || Some(b) == lowest_normal)) { continue; }`. A higher-bucket candidate is therefore never redeemed, so no lower non-empty bucket can be bypassed. `first_set` (find-first-set) is at `crates/fusd-math/src/rate_bucket.rs:52-62`.

**Formal verification.** The `first_set` primitive is proptest-covered (**B8** `rate_bucket`, incl. `first_set_from` partial-word masking) and unit-tested (`crates/fusd-math/src/rate_bucket.rs:157-193`). But the **instruction-level** "starts at the lowest bucket" property has **no FV artifact yet**: `certora/mutations.md` row **B3** (`redeem_targets_lowest_bucket_and_preserves_coherence`, mutation: accept candidates from an arbitrary bucket) is explicitly flagged ⚠ NOT YET COVERED — no Certora rule and no redeem targeting test. Enforced only at the instruction level (primitive proven; instruction-level assertion pending).

## 5. Rejected Alternatives

### Not algorithmic-seigniorage — overcollateralized CDP instead
Every fUSD is minted only against collateral worth strictly more than the debt at the pessimistic-LOW `spot`; there is no uncollateralized/rebasing supply lever. Structurally enforced at mint: `borrow.rs:107` `require!(cdp::is_healthy(ink, new_debt, spot, mcr), FusdError::BelowMinCollateralRatio)`, where `is_healthy` = `recorded_debt <= max_debt(ink·spot, mcr_bps)` (`programs/fusd-core/src/cdp.rs:48`). Philosophy: `.internal/ARCHITECTURE.md:9`.

### No PSM (Peg-Stability Module)
No 1:1 stable-for-fUSD swap facility exists — that would import the swapped stable's risk as un-isolated shared backing and undercut credible neutrality. The peg rests instead on the four code loops (redemption floor, repay-arbitrage, Reactor Pool, user-set rates): `.internal/ARCHITECTURE.md:16`. No PSM instruction is present in the instruction set (`programs/fusd-core/src/instructions/mod.rs`).

### Not a shared-collateral basket — per-market isolation
Each collateral is its own market/vault/position lane; no basket co-mingles backing. Enforced by PDA sharding: `Market` seeded `[MARKET_SEED, collateral_mint]` (`programs/fusd-core/src/instructions/init_market.rs:54`) with its own escrow vault (`:62`), `Position` seeded `[position, collateral_mint, owner]` (`.internal/ARCHITECTURE.md:90`). The global-backstop guardrail permits protocol-owned fUSD flows only — never collateral/risk between user pools: `.internal/ARCHITECTURE.md:230`.

### No Recovery Mode — static `health < MCR` liquidation (decision #6)
Liquidation eligibility never depends on system/market state, so a crash can't expand the liquidatable set and deepen the spiral. The rule is the static `liquidate.rs:141` `require!(!cdp::is_healthy(ink, debt, liq_spot, mcr), FusdError::PositionHealthy)` — no TCR/price-of-others term. Rationale: `.internal/ARCHITECTURE.md:228`. (`cdp::tcr_below` exists only for shutdown-SCR + the CCR borrow band, never to widen liquidation — `programs/fusd-core/src/cdp.rs:66`.)

### No Token-2022 — legacy SPL only, rejected by typing
Token-2022's censorship extensions (PermanentDelegate/TransferHook/Pausable/DefaultAccountState/etc.) are excluded by construction: the collateral mint is `Box<Account<'info, Mint>>` validated under `Program<'info, Token>` (legacy `Tokenkeg…`), so a T22 mint fails with `AccountOwnedByWrongProgram` before the handler runs — `programs/fusd-core/src/instructions/init_market.rs:48,78`. The one legacy hazard, a freeze authority, is rejected explicitly: `:89` `require!(ctx.accounts.collateral_mint.freeze_authority.is_none(), FusdError::CollateralHasFreezeAuthority)`. Onboarding gate: `.internal/ARCHITECTURE.md:260`; fUSD mint itself is legacy with `freeze_authority = None`: `.internal/ARCHITECTURE.md:255-258`.

### No global Vat — account-sharded, Sealevel-parallel
Maker's single global `Vat` would serialize every state change behind one write-lock/CU lane; fUSD shards state per collateral and per user so distinct collaterals and distinct users never contend. Rejection + rationale: `.internal/ARCHITECTURE.md:13,84`; the hot path writes only `Position` + `Market` (`.internal/ARCHITECTURE.md:101`), realized by the per-collateral `Market` / per-`[mint,owner]` `Position` PDAs (`programs/fusd-core/src/instructions/init_market.rs:54`).

### No discretionary admin pause — rule-based per-market shutdown + monotonic guardian
There is deliberately no global emergency flag: `ProtocolConfig` carries none (`programs/fusd-core/src/state/protocol_config.rs:54-56`, removed 2026-06-12 klend-C13; `.internal/ARCHITECTURE.md:88`). The only levers are (a) permissionless per-market `shutdown`, triggerable ONLY on rule conditions — `shutdown.rs:59` `require!(oracle_failed || tcr_breach, FusdError::ShutdownConditionNotMet)`, "no discretionary trigger" (`shutdown.rs:5`); and (b) a guardian that can only pause NEW borrowing, auto-lifting and clamped — `guardian_derisk.rs:48` `require!((0..=GUARDIAN_MAX_PAUSE_SECS).contains(&pause_secs), …)`, `:57` sets `guardian_paused_until = now + pause_secs`; it cannot seize/freeze/mint/raise-MCR/impede-redemption (`guardian_derisk.rs:8-16`). Governance bounds: `.internal/ARCHITECTURE.md:274`.

### No auction liquidation — Liquity offset/redistribute
Collateral auctions failed catastrophically under congestion + oracle lag (Maker Black Thursday: 1,462 zero-bid wins, 5.67M DAI bad debt), so fUSD uses O(1) permissionless offset/redistribute tiers instead. Rationale + tiers: `.internal/ARCHITECTURE.md:224-227,230`; the strict-order waterfall (Reactor Pool offset → redistribution → local insurance buffer → global backstop → un-homed) is `crates/fusd-math/src/recovery.rs:64` `absorb` (conservation `reactor + redist + buffer + global + unhomed == debt`), invoked at `liquidate.rs:223` with the RP offset applied at `liquidate.rs:256` — no bidding, no keeper price discovery.
