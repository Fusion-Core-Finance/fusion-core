# fusd-math — Kani proof-strength audit

Companion to the 20 `#[kani::proof]` harnesses in [`src/kani_proofs.rs`](src/kani_proofs.rs). It
records, for each harness, **how strong the proof actually is** — because a Kani proof can report
`VERIFICATION SUCCESSFUL` while proving almost nothing (an unreached branch, a vacuous assertion, a
concrete-input collapse). This is the auditor-facing strength table for the pre-audit
formal-verification tier (fusion-docs.md).

## How to run / gate

```text
cargo kani -p fusd-math          # verify all 20 harnesses (whole suite, ~6 min)
scripts/kani-audit.sh            # isolated runner: each harness alone (cbmc killed between), then
                                 #   regenerate kani_audit.tsv and GATE (fail on any non-PASS /
                                 #   TIMEOUT / untagged / VACUOUS harness)
scripts/kani-audit.sh --gate     # fast gate only (no Kani run): re-check the source strength tags
                                 #   and assert the committed kani_audit.tsv shows PASS for every harness
cargo test -p fusd-math          # the always-on differential tests (the primary correctness guarantee)
```

The strength tag is a `// strength: <TAG> — <justification>` line directly above each `#[kani::proof]`
in `src/kani_proofs.rs` — the **single source of truth** the runner and gate read.
[`kani_audit.tsv`](kani_audit.tsv) is the tracked run artifact (regenerated by the runner).

## The taxonomy

- **INDUCTIVE** — `∀ symbolic state. INV(s) ⟹ INV(f(s))`, fully symbolic initial state, decomposed
  invariants. The gold standard. **Not applicable here**: these are pure-function value/rounding/
  conservation proofs, not state-machine transition proofs, so none qualifies (and none claims it).
- **STRONG** — symbolic inputs exercise all *reachable* branches, a real (un-weakened) property is
  asserted, non-vacuous (a `kani::cover!` forces the interesting case). A *deliberate, disclosed*
  tractability bound (a concrete pool-size divisor `REACTOR_POOL = 97`, or `u8`/`<16` narrowing because the
  property is scale-independent) is acceptable as STRONG **only** when the missing generality is
  genuinely covered elsewhere by an always-on differential test.
- **WEAK** — symbolic but misses a reachable branch, or asserts a weaker property than warranted.
- **UNIT_TEST** — concrete-only inputs / single execution path (still valuable, but labelled honestly).
- **VACUOUS** — assertions may be unreachable (a real defect — the gate **rejects** this tag).

## Methodology

Each harness was classified independently (three module-grouped classifiers), then an adversarial
reviewer re-checked every tag against the source, defaulting to the lower tag on doubt. Result:
**0 downgrades, 0 VACUOUS** — the tags below are the adversarially-confirmed labels.

### Per-position interest (`src/interest.rs`)

The BOLD weighted-debt-sum accrual. The denominator `SECONDS_PER_YEAR·10_000` is a **compile-time
constant**, so the divides stay cheap; a **concrete `SECONDS_PER_YEAR` period** makes the year factor
cancel exactly (`floor(d·r·Y/(Y·10_000)) == floor(d·r/10_000)`), pinning the formula/units/rounding
non-vacuously over a ≤16-bit symbolic `(debt, rate)`. Arbitrary-period generality is swept by the
always-on `matches_u256_reference` differential test (20k inputs vs a U256 reference).

| Harness | Strength | Proves | Caveat / where the rest is covered |
|---|---|---|---|
| `accrued_one_year_is_debt_times_rate` | STRONG | per-position interest == `floor(debt·rate/10_000)` at the year boundary (formula, units, FLOOR direction) | symbolic `u8` debt/rate; arbitrary period swept by `matches_u256_reference` |
| `pending_aggregate_one_year_rounds_up` | STRONG | aggregate interest == `ceil(weighted/10_000)` — the round-UP the mint depends on | symbolic `u16` weighted; wide sweep in `matches_u256_reference` |
| `aggregate_never_short_of_position` | STRONG | **solvency**: aggregate ceil ≥ position floor for the same weight, differ ≤ 1 (the no-drift margin) | concrete year; the relation is structural (`ceil ≥ floor` of one numerator) |
| `accrued_zero_at_boundaries` | STRONG | interest == 0 at each boundary (no time / no rate / no debt) | fully symbolic small inputs; unconditional asserts → structurally non-vacuous |
| `accrued_fails_closed_on_overflow` | UNIT_TEST | the weighted-term overflow fails closed (`None`, no wrap) | concrete witnesses; the symbolic over-`u128` sweep is in `matches_u256_reference` |

## Summary

**STRONG ×21 · WEAK ×2 · UNIT_TEST ×2 · VACUOUS ×0** (25 harnesses). Every STRONG harness asserts a real
(un-weakened) value / rounding-direction / conservation property and carries a `kani::cover!` forcing
its interesting case (the lone exception, `mul_div_zero_denominator_is_none`, is non-vacuous
structurally). Each discloses its tractability bound, and the missing wide-divide / loop / arbitrary-period
generality is genuinely covered by the **always-on differential tests** — `shim_matches_bnum`,
`muladd_div_matches_bnum`, `update_product_matches_bnum`, `ray_pow_matches_reference`, and
`interest::matches_u256_reference` (20k full-range inputs each) — which are the primary correctness
guarantee for the shimmed/looped/divided paths, independent of any Kani run. The two WEAK harnesses are
honestly weak (`ray_pow_identities` proves only the trivial `x^0`;
`compute_stake_identity_before_first_liquidation` is a genesis pass-through), and each has a STRONG
sibling or a differential test covering the substantive path. The two UNIT_TEST harnesses
(`mul_div_fails_closed_on_overflow`, `accrued_fails_closed_on_overflow`) are the correct label for
all-concrete overflow-witness sets.

## proptest coverage (B8 — the wide-random / stateful complement)

The Kani harnesses above are **exhaustive over tiny, bit-narrowed inputs** (the tractability bound each
discloses). The B8 pass adds **`proptest`** property suites — in each module's `#[cfg(test)]` block — that
assert the **same** value / rounding-direction / conservation properties over **wide random inputs** and,
the part Kani structurally can't reach, **random operation *sequences*** (stateful fuzzing). The three
verification layers are complementary:

- **Kani** — *every* small value, exhaustively (rounding direction, conservation, fail-closed).
- **proptest** — *thousands of* wide random values **and random op-sequences** (the bitmap concurrent-flip;
  the RP offset sequence across scale/epoch rolls) — what bounded symbolic proof can't enumerate.
- **differential tests** (`shim_matches_bnum`, `update_product_matches_bnum`, `ray_pow_matches_reference`,
  `interest::matches_u256_reference`, …) — 20k full-range inputs pinning the shimmed / looped / divided paths.

| Module | proptests | What proptest adds beyond Kani |
|---|---|---|
| `rate_bucket` | 14 | the **stateful concurrent-flip** `BTreeSet` model; `first_set_from` partial-word masking over random `k`; `cmp_collateral_ratio` order laws (anti-symmetry / transitivity / scale-invariance / independent ref) |
| `reactor_pool` | 7 | **stateful offset sequences** across scale/epoch rolls; wide solvency + collateral conservation; the **depositor-snapshot round-trip** carrying an entry snapshot across scale bumps + the epoch roll |
| `oracle_scale` | 14 | monotonicity; **constructive overflow→`None`** (fail-closed) for all three fns; full-U256 `px_to_ray` exact reference; Q64.64 decode within a ~`1e-12` relative band |
| `interest` | 6 | round-up + aggregate-never-short over wide `(debt, rate, period)` vs a U256 reference |
| `redistribution` | 6 | conservation + **no-drift over 400 redistributions** (discriminating non-round 1e16–1e18 stakes) + stake bounds |
| `recovery` | 3 | waterfall conservation + fail-closed/ordered over wide u128 |

**Most** proptests check an **independent reference** (a `BTreeSet`, a naive U256 / f64-with-tolerance
recompute, a direct sum) computed a *different* way than the code under test. A few
(`bucket_of_matches_reference`, `s_idx_matches_reference`, `absorb_matches_naive_reference`) instead
compare against a deliberately-simpler **re-derivation of the same formula** — these pin the
formula/units and guard against refactor drift rather than catching a logic *divergence*, and the
substantive direction (rounding, conservation, fail-closed) is covered for those functions by the
structural properties here plus their Kani harnesses. Every generator respects its function's
precondition (out-of-contract `None`/`Err` is correct behavior, not a counterexample). Run via
`cargo test -p fusd-math`. **The B8 sweep found no counterexamples** — every property held.

### Overflow envelope (BOLD-sweep C4)

BOLD's README "known issue 15" makes recomputing the RP overflow threshold a **mandatory checklist item
for any fork with a different precision or supply**. fUSD changed both axes vs Liquity (`u128` not
`uint256`; 6-decimal native units not 18), so the analysis is redone and pinned. Full derivation in the
`reactor_pool` / `redistribution` module docs; the load-bearing facts:

- **Revert, never wrap.** Every overflow path returns an error (`ReactorError::Math` / `ScaleOverflow` /
  `EpochOverflow`; `RedistError::AccumulatorOverflow`) — a known migration trigger, never a silent loss.
- **Bounds far above any plausible per-market supply.** The absolute per-offset edge is
  `total/debt/coll ≤ u128::MAX / 1e18 ≈ 3.40e20` native = **~$340T** at 6 decimals (a $10B market is
  1e16 native — ~34,000× margin). The marginal `coll_gain_per_unit·P` edge is the ratio
  `coll/total ≤ ~340`; the realized SOL/LST ratio is ≈ 6.7 (~50× margin). The `[128 epoch × 64 scale]`
  grid absorbs 127 catastrophic full-drain epoch rolls before the migration trigger.
- **Pinned by three unit regressions** (`reactor_pool`): `large_realistic_market_with_lst_ratio_never_overflows`
  (a $10B market at the realized ~7× ratio — the `1 < ratio ≪ 340` regime the proptests, bounded to
  `coll ≤ total`, skip — never overflows), `epoch_grid_absorbs_max_rolls_then_reverts` (absorbs the max
  rolls, then `EpochOverflow`), `offset_reverts_not_wraps_beyond_envelope` (all three multiply sites
  revert past the envelope); plus `redistribute_reverts_not_wraps_beyond_envelope` (`redistribution`).

### Certora follow-on (the B8 → Phase-1-exit path)

Kani + proptest cover the **pure-function** math in `fusd-math`. What neither reaches is the
**cross-instruction, multi-account** invariants of the live program — properties that hold only across
*sequences of real transactions over the on-chain accounts*, which is **Certora's** domain (CVLR over the
`fusd-core` instructions). The four properties that map the Phase-1 formal-verification exit, with their
**current Certora status** (the harness now exists — see the coverage table below):

1. **Global supply invariant** — `circulating fUSD == agg_recorded_debt − unminted_interest + bad_debt` across
   every `borrow`/`repay`/`liquidate`/`redeem`/`refresh_market` sequence (the per-tx pieces are unit/property
   tested; the cross-tx ledger identity is Certora's). **`borrow` VERIFIED (cloud); the six sibling
   instructions authored + wired, pending a cloud run.**
2. **On-chain bitmap concurrent-flip** — the rate-bucket bit math is proptest-proven here, but the program-level
   "no two concurrent redemptions disagree on the lowest non-empty bucket" rests on the per-market `Market`
   write-lock serialization — a cross-transaction property, not a pure-function one. **VERIFIED (cloud)** at
   the `add_member`/`remove_member` coupling level (Sealevel's per-market write-lock supplies the
   serialization; Certora proves each touch atomically preserves coupling).
3. **Liquidation always terminates** — every liquidation routes into exactly one of {RP offset, redistribution,
   buffer, shutdown}. The `recovery` waterfall is proven pure here; the *instruction* must always reach it.
   **VERIFIED (cloud)**: full-debt conservation + strict tier ordering over symbolic `recovery::absorb`.
4. **RP `P`/`S` consistency** across `provide`/`withdraw`/`offset` instruction interleavings — a depositor can
   always realize their gain; no silent loss across the real account lifecycle. **DEFERRED** — the pool's
   `bnum` U256 division is intractable for the SMT backend (times out); stays covered by Kani + proptest +
   `integration-tests/litesvm_reactor_realizability`.

B8 (this pass) clears the math layer beneath these; Certora is the cross-instruction FV tier above it.

### Mutation coverage (cargo-mutants — measuring what the *suite* actually constrains)

Kani and proptest say what the math *proves*; mutation testing says what the *test suite* would *notice*.
`cargo-mutants` mechanically breaks every line of the crate (operator swaps, comparison flips, return-value
replacements) and reruns the suite against each — a surviving mutant is a line no test constrains. This is
the whole-crate, automated form of the four manual mutation checks the B8 review ran by hand.

**Config (committed, reproducible):** `crates/fusd-math/mutants.toml` + `scripts/mutants-fusd-math.sh`. The
wrapper documents the two-stage proptest-determinism method: a broad sweep at the suite's tuned in-code case
counts, then high-`PROPTEST_CASES` confirmation of every survivor (cargo-mutants runs each suite once, and
the B8 proptests use an unpinned seed, so a single low-case run can flag a probabilistic catch as a false
survivor). `kani_proofs.rs` is excluded (`#[cfg(kani)]`, never built under `cargo test`).

**Result:** 377 mutants generated → **2 unviable** (whole-function `Default::default()` replacements that
don't compile) · **375 viable**. Of the 375: **361 caught = every non-equivalent mutant (100%)**; the
remaining **14 are justified equivalent mutants** (behaviourally inert, excluded by exact description in
`mutants.toml` with a one-line rationale each).

The first sweep left 23 survivors; triage split them into **9 real coverage gaps** (now killed by new
tests, each shown to flip the mutant from survived→caught — the B8 loop) and **14 equivalent mutants**:

| Real gap (now killed) | New test | Property pinned |
|---|---|---|
| `wad_mul_up` fully replaceable (`Some(0)`/`None`/`Some(1)`) — the WAD ceil twin was untested (prod uses `ray_mul_up`) | `wad_ray_ops` (extended) | ceil rounds up by 1 on a remainder |
| `offset` error-feedback writeback (`- → /`) — `partial_offset_stays_solvent` only bounds `last_loss_error ∈ [1,total]` | `offset_pins_exact_loss_error_feedback` | exact residual `total − (debt·1e18 % total)` |
| `compounded_deposit` scale-diff (`- → +`, `==1 → !=1`) — offset/Kani suites run only at scale 0 | `compounded_deposit_across_scale_roll` | per-scale `/SCALE_FACTOR`, hard-0 at ≥2 rolls |
| `compounded_deposit` dust guard (`< → <=`, `< → ==`) | `compounded_deposit_dust_guard_at_threshold` | strict `<`: on-floor deposit kept |
| `collateral_gain` next-scale portion (`/ → %`) | `collateral_gain_includes_next_scale_portion` | `snap.scale+1` cell folded in, `/SCALE_FACTOR` |

The scale-roll gaps are the notable ones: the RP `P`/`S` scale-bump arithmetic was unexercised by **both**
proptest **and** Kani (both feed `u8` inputs that never bump `scale`), so the depositor-realizability-across-
a-scale-roll path was unconstrained by the suite. The code is correct Liquity logic — these are coverage
gaps now closed, not bugs. (`wad_mul_up`/`HALF_WAD`/`HALF_RAY` surfaced as unused dead `pub` items — flagged,
not removed.)

The 14 equivalents, by class: unused dead consts (`HALF_WAD`/`HALF_RAY`); a `#[cfg(kani)]`-only line; two
disjoint-bit `| ≡ ^` ORs; an `expo==0` boundary where `base·1 == base/1`; a `|| ≡ &&` early-out that
`mul_div_floor(0,·)=0` makes inert; a `bumps>0 ≡ >=0` no-op `+0`; and the unreachable `bumps>4` rescale cap
(the loop provably terminates in ≤2 bumps). Full per-mutant rationale: `mutants.toml` `exclude_re`.

This completes the four-tier FV-evidence picture for `fusd-math`: **Kani** (exhaustive over tiny inputs) ·
**proptest** (wide random / stateful) · **cargo-mutants** (suite-strength: 100% of non-equivalent mutants
killed) · **Certora** (cross-instruction, above the crate).
