Companion to the 20 `#[kani::proof]` harnesses in [`src/kani_proofs.rs`](src/kani_proofs.rs). It
records, for each harness, **how strong the proof actually is** â€” because a Kani proof can report

## How to run / gate

```text
cargo kani -p fusd-math          # verify all 20 harnesses (whole suite, ~6 min)
scripts/kani-audit.sh            # isolated runner: each harness alone (cbmc killed between), then
                                 #   regenerate kani_audit.tsv and GATE (fail on any non-PASS /
                                 #   TIMEOUT / untagged / VACUOUS harness)
scripts/kani-audit.sh --gate     # fast gate only (no Kani run): re-check the source strength tags
                                 #   and assert the committed kani_audit.tsv shows PASS for every harness
cargo test -p fusd-math          # the always-on differential tests (the primary correctness guarantee)
```

The strength tag is a `// strength: <TAG> â€” <justification>` line directly above each `#[kani::proof]`
in `src/kani_proofs.rs` â€” the **single source of truth** the runner and gate read.
[`kani_audit.tsv`](kani_audit.tsv) is the tracked run artifact (regenerated by the runner).

## The taxonomy

- **INDUCTIVE** â€” `âˆ€ symbolic state. INV(s) âŸ¹ INV(f(s))`, fully symbolic initial state, decomposed
  invariants. The gold standard. **Not applicable here**: these are pure-function value/rounding/
  conservation proofs, not state-machine transition proofs, so none qualifies (and none claims it).
- **STRONG** â€” symbolic inputs exercise all *reachable* branches, a real (un-weakened) property is
  asserted, non-vacuous (a `kani::cover!` forces the interesting case). A *deliberate, disclosed*
  tractability bound (a concrete pool-size divisor `REACTOR_POOL = 97`, or `u8`/`<16` narrowing because the
  property is scale-independent) is acceptable as STRONG **only** when the missing generality is
  genuinely covered elsewhere by an always-on differential test.
- **WEAK** â€” symbolic but misses a reachable branch, or asserts a weaker property than warranted.
- **UNIT_TEST** â€” concrete-only inputs / single execution path (still valuable, but labelled honestly).
- **VACUOUS** â€” assertions may be unreachable (a real defect â€” the gate **rejects** this tag).

## Methodology

Each harness was classified independently (three module-grouped classifiers), then an adversarial
reviewer re-checked every tag against the source, defaulting to the lower tag on doubt. Result:
**0 downgrades, 0 VACUOUS** â€” the tags below are the adversarially-confirmed labels.

## proptest coverage (B8 â€” the wide-random / stateful complement)

The Kani harnesses above are **exhaustive over tiny, bit-narrowed inputs** (the tractability bound each

### Overflow envelope (BOLD-sweep C4)

BOLD's README "known issue 15" makes recomputing the RP overflow threshold a **mandatory checklist item
for any fork with a different precision or supply**. fUSD changed both axes vs Liquity (`u128` not
`uint256`; 6-decimal native units not 18), so the analysis is redone and pinned. Full derivation in the
`reactor_pool` / `redistribution` module docs; the load-bearing facts:

- **Revert, never wrap.** Every overflow path returns an error (`ReactorError::Math` / `ScaleOverflow` /
  `EpochOverflow`; `RedistError::AccumulatorOverflow`) â€” a known migration trigger, never a silent loss.
- **Bounds far above any plausible per-market supply.** The absolute per-offset edge is
  `total/debt/coll â‰¤ u128::MAX / 1e18 â‰ˆ 3.40e20` native = **~$340T** at 6 decimals (a $10B market is
  1e16 native â€” ~34,000Ã— margin). The marginal `coll_gain_per_unitÂ·P` edge is the ratio
  `coll/total â‰¤ ~340`; the realized SOL/LST ratio is â‰ˆ 6.7 (~50Ã— margin). The `[128 epoch Ã— 64 scale]`
  grid absorbs 127 catastrophic full-drain epoch rolls before the migration trigger.
- **Pinned by three unit regressions** (`reactor_pool`): `large_realistic_market_with_lst_ratio_never_overflows`
  (a $10B market at the realized ~7Ã— ratio â€” the `1 < ratio â‰ª 340` regime the proptests, bounded to
  `coll â‰¤ total`, skip â€” never overflows), `epoch_grid_absorbs_max_rolls_then_reverts` (absorbs the max
